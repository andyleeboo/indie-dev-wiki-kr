# 자동화 (Automation)

## 개요

인디 개발자는 리소스가 제한적이므로 반복 업무·빌드·테스트·배포 같은 작업을 자동화하는 것이 중요하다. 자동화는 개발 속도를 높이고 오류를 줄이며 일관된 품질을 보장한다.

## 1. 자동화란?

자동화는 수동으로 반복하던 작업을 코드나 툴로 대신 수행하게 만드는 것이다. 빌드, 테스트, 배포, 모니터링뿐 아니라 AI를 통한 루틴 업무 처리까지 포함될 수 있다.

## 2. CI/CD

CI/CD는 코드를 통합하고 자동으로 테스트 및 배포하는 파이프라인이다.

### 목표

- 변경 사항이 정기적으로 통합되도록
- 자동 테스트
- 빠른 배포

### 예시

- GitHub Actions
- GitLab CI

## 3. 테스트 자동화

- 단위 테스트, 통합 테스트
- 자동 보고서 생성
- 배포 전 에러 사전 감지

## 4. AI 기반 자동화

AI를 이용하면 반응형 사용자 피드백 분석, 로그 에러 분류, 코드 품질 자동 측정 등도 자동화할 수 있다.

## 5. 생산성 자동화 툴

- **Zapier, Make.com**: 외부 서비스 연결 자동화
- **No-code 플랫폼**: 반복적 UI/UX 워크플로우 생략
- **스크립트 기반 자동 빌드/배포**

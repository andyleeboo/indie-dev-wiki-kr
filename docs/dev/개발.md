# 개발 (Development)

## 개요

개발 단계는 아이디어를 실제 동작하는 코드로 옮겨 MVP(Minimum Viable Product)를 만드는 과정이다. 단순히 기능을 구현하는 것뿐 아니라 확장성, 유지보수, 비용을 고려한 아키텍처 설계가 필요하다.

## 1. MVP란?

MVP는 최소한의 기능만 포함한 프로토타입 버전으로, 빠르게 출시해 사용자 반응을 수집하는 데 목적이 있다. 이 접근법은 Lean Startup 철학에서 중요하게 여겨진다.

## 2. MVP 개발 전략

### 1) 기능 최소화

핵심 가치만 담은 기능으로 빠르게 출시한다.

### 2) 반복 개선

사용자의 반응과 데이터를 기반으로 기능을 반복적으로 다듬는다.

### 3) 데이터 중심

로그, 에러 추적, 분석을 통해 사용자 행동을 파악한다.

### 4) 스케일 준비

확장성, 성능, 보안 등을 고려하여 설계한다.

## 3. AI 도구와 개발

AI는 코드 생성, 테스트 자동화, 설계 도출 등 개발 전반을 가속화한다. 이를 통해 빠른 프로토타입 제작과 반복 개발이 가능해졌다.

### 주요 활용 사례

- 코드 생성/자동완성 (예: Copilot)
- 테스트 자동화
- 설계 대안 제안
- 타깃 사용자 성향 예측
- 문서/스펙 자동 생성

## 4. 기술 스택 선택

기술 스택은 프로젝트 목적, 팀 능력, 확장성에 따라 다르다.

### 예시

- 프론트엔드: React, Vue, Svelte
- 백엔드: Node.js, Django, Go
- DB: PostgreSQL, MongoDB
- 배포: Vercel, Netlify, AWS

## 5. 개발 워크플로우

- 버전 관리: Git
- 코드 리뷰/협업: GitHub PR
- CI/CD: GitHub Actions
- 이슈/태스크: Jira/Notion

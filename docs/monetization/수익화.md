# 수익화

인디 개발자를 위한 앱/게임 수익화 전략 가이드입니다. 이 문서는 다양한 수익화 모델과 각각의 장단점, 구현 방법, 그리고 실전에서 유용한 팁을 다룹니다.

## 광고 수익

광고는 무료 앱에서 가장 일반적인 수익화 방법입니다. 사용자에게 앱을 무료로 제공하면서도 수익을 창출할 수 있습니다.

### AdMob

구글의 AdMob은 모바일 앱 광고 플랫폼으로, Android와 iOS 모두를 지원합니다.

**주요 광고 형식:**
- 배너 광고 (Banner Ads): 화면 상단 또는 하단에 고정
- 전면 광고 (Interstitial Ads): 전체 화면을 차지하는 광고
- 보상형 광고 (Rewarded Ads): 광고 시청 후 게임 내 보상 제공
- 네이티브 광고 (Native Ads): 앱의 디자인에 자연스럽게 통합

**장점:**
- Firebase와의 통합으로 쉬운 구현
- 높은 fill rate와 안정적인 수익
- 다양한 광고 형식 지원

**단점:**
- 구글 정책 위반 시 계정 정지 위험
- 사용자 경험 저하 가능성

### AdSense

주로 웹 기반 콘텐츠에 사용되지만, 웹뷰를 활용한 앱에서도 사용 가능합니다.

**특징:**
- 웹 개발자에게 익숙한 인터페이스
- 블로그나 웹 서비스와 연계 시 효과적
- 모바일 앱보다는 웹 콘텐츠에 최적화

### 광고 네트워크 비교

| 네트워크 | eCPM | Fill Rate | 지원 플랫폼 | 특징 |
|---------|------|-----------|------------|------|
| AdMob | 중상 | 높음 | iOS, Android | Firebase 통합, 다양한 광고 형식 |
| Unity Ads | 중 | 높음 | iOS, Android, Unity | 게임에 특화, 보상형 광고 강점 |
| AppLovin | 상 | 중상 | iOS, Android | 높은 eCPM, MAX 중재 플랫폼 제공 |
| ironSource | 상 | 높음 | iOS, Android | 하이퍼캐주얼 게임에 강점 |
| Facebook Audience Network | 중상 | 중 | iOS, Android | 타겟팅 정확도 높음 |

### 광고 수익 최적화 팁

**미디에이션 활용:**
여러 광고 네트워크를 동시에 사용하여 fill rate를 높이고 eCPM을 최적화할 수 있습니다. AdMob 미디에이션 또는 AppLovin MAX 같은 플랫폼을 활용하세요.

**광고 배치 전략:**
- 보상형 광고는 게임 내 자연스러운 포인트에 배치 (레벨 클리어 후, 추가 생명 획득 등)
- 전면 광고는 사용자 플로우를 방해하지 않는 지점에 배치
- 배너 광고는 핵심 UI 요소를 가리지 않도록 주의

**A/B 테스팅:**
광고 빈도, 배치, 형식을 다양하게 테스트하여 수익과 사용자 유지율의 최적 균형점을 찾으세요.

## 인앱결제 (IAP)

인앱결제는 앱 내에서 디지털 상품이나 서비스를 판매하는 수익화 방법입니다.

### iOS (StoreKit)

Apple의 StoreKit 프레임워크를 통해 구현됩니다.

**제품 유형:**
- 소비성 제품 (Consumable): 게임 화폐, 아이템 등 반복 구매 가능
- 비소비성 제품 (Non-Consumable): 광고 제거, 프리미엄 기능 등 영구 소유
- 자동 갱신 구독 (Auto-Renewable Subscription): 정기 구독 서비스
- 비갱신 구독 (Non-Renewing Subscription): 제한된 기간의 구독

**주요 특징:**
- App Store의 30% 수수료 (구독 1년 이후 15%)
- 샌드박스 환경에서 테스트 가능
- 영수증 검증 필수 (서버 측에서 수행 권장)
- StoreKit 2 도입으로 async/await 지원 (iOS 15+)

**구현 팁:**
- 영수증 검증은 반드시 서버에서 수행하여 해킹 방지
- 복원 기능 (Restore Purchases) 필수 구현
- 구매 프로세스 중 네트워크 오류 처리에 주의
- App Store Connect에서 제품 등록 시 현지화된 설명 추가

### Android (Google Play Billing)

Google Play Billing Library를 통해 구현됩니다.

**제품 유형:**
- 일회성 제품 (One-time Products): 소비성/비소비성 모두 포함
- 구독 (Subscriptions): 자동 갱신 구독

**주요 특징:**
- Google Play의 30% 수수료 (구독 1년 이후 15%)
- Billing Library 버전 관리 중요 (현재 최신: BLL 6.x)
- 실시간 개발자 알림 (RTDN) 활용 가능
- Play Console에서 테스트 계정 설정 가능

**구현 팁:**
- BillingClient 생성 및 연결은 앱 시작 시 수행
- PurchasesUpdatedListener로 구매 완료 처리
- acknowledgePurchase 또는 consumeAsync 호출 필수 (3일 내 미처리 시 환불)
- Pending Transactions 처리 로직 구현
- ProGuard 설정 시 Billing Library 클래스 제외

### iOS와 Android 차이점

| 항목 | iOS (StoreKit) | Android (Play Billing) |
|------|----------------|------------------------|
| 제품 유형 구분 | 4가지 명확한 구분 | 2가지 (일회성/구독) |
| 영수증 검증 | 영수증 기반 | 토큰 기반 |
| 구매 확인 | 자동 완료 | 명시적 acknowledge 필요 |
| 테스트 | 샌드박스 계정 | 테스트 계정 + 라이센스 테스터 |
| 복원 | 별도 API 호출 | queryPurchasesAsync로 조회 |
| 수수료 | 30% (구독 1년 후 15%) | 30% (구독 1년 후 15%) |

### 크로스 플랫폼 구현

Unity, Flutter, React Native 등을 사용하는 경우 다음 플러그인을 활용할 수 있습니다:

- **Unity**: Unity IAP 패키지
- **Flutter**: in_app_purchase 패키지
- **React Native**: react-native-iap

모든 경우에 서버 측 영수증 검증을 구현하는 것이 보안상 중요합니다.

## 구독 모델

구독 모델은 안정적이고 예측 가능한 수익을 창출할 수 있는 방법입니다.

### 장점

**안정적인 수익:**
월간 반복 수익(MRR)으로 비즈니스 예측 가능성이 높아집니다.

**높은 생애 가치 (LTV):**
장기 구독자는 일회성 구매보다 훨씬 높은 총 수익을 창출합니다.

**지속적인 관계:**
사용자와 장기적인 관계를 형성하고 피드백을 받을 수 있습니다.

**플랫폼 혜택:**
iOS와 Android 모두 1년 이후 수수료가 15%로 감소합니다.

### 단점

**초기 진입 장벽:**
사용자가 구독을 시작하기까지의 심리적 장벽이 높습니다.

**이탈 관리:**
매월 이탈을 방지하기 위한 지속적인 가치 제공이 필요합니다.

**지속적인 개발 필요:**
구독자를 유지하기 위해 정기적인 업데이트와 새로운 기능 추가가 필수입니다.

**환불 및 취소 처리:**
구독 취소, 결제 실패, 환불 등의 케이스를 처리해야 합니다.

### 가격 전략

**다층 가격 책정:**
여러 구독 계층을 제공하여 다양한 사용자 니즈를 충족시킵니다.

예시:
- Basic: 월 $2.99 - 기본 기능
- Pro: 월 $5.99 - 고급 기능
- Premium: 월 $9.99 - 모든 기능 + 우선 지원

**연간 구독 할인:**
월간 구독 대비 20-30% 할인된 연간 구독 옵션을 제공하여 장기 커밋을 유도합니다.

**무료 체험:**
3일, 7일, 14일 또는 30일 무료 체험을 제공하여 사용자가 가치를 경험하게 합니다. iOS에서는 Introductory Offers를 활용할 수 있습니다.

**지역별 가격 책정:**
구매력 평가(PPP)를 고려한 지역별 차등 가격을 설정합니다.

**프로모션 코드:**
특정 사용자 그룹이나 이벤트를 위한 할인 쿠폰을 활용합니다.

### 구독 최적화 팁

**온보딩 개선:**
사용자가 앱의 핵심 가치를 빠르게 이해하도록 온보딩을 최적화합니다.

**이탈 방지:**
구독 취소 전 피드백 수집, 할인 제안, 또는 다운그레이드 옵션을 제공합니다.

**리텐션 집중:**
신규 구독자 확보보다 기존 구독자 유지가 비용 효율적입니다. Cohort 분석으로 이탈 시점을 파악하세요.

**Grace Period 활용:**
결제 실패 시 유예 기간을 제공하여 실수로 인한 이탈을 방지합니다.

## 프리미엄/유료 앱 전략

선불 모델로, 사용자가 앱을 다운로드하기 전에 비용을 지불합니다.

### 프리미엄 모델의 장점

**즉시 수익:**
다운로드 즉시 수익이 발생합니다.

**광고 없는 경험:**
광고 없이 깔끔한 사용자 경험을 제공할 수 있습니다.

**명확한 가치 제안:**
한 번의 결제로 모든 기능을 제공하는 단순한 모델입니다.

### 프리미엄 모델의 단점

**낮은 다운로드 수:**
무료 앱 대비 다운로드 수가 현저히 낮습니다.

**마케팅 어려움:**
사용자가 경험해보기 전에 결제해야 하므로 신뢰 구축이 어렵습니다.

**환불 압력:**
기대에 미치지 못할 경우 환불 요청이 많을 수 있습니다.

**제한적인 성장:**
네트워크 효과나 바이럴 성장이 어렵습니다.

### Freemium vs Premium

많은 인디 개발자에게 Freemium (무료 + 인앱결제) 모델이 Premium보다 효과적입니다:

**Freemium의 장점:**
- 높은 다운로드 수로 사용자 기반 확보
- 사용자가 앱을 경험한 후 결제하므로 전환율 향상
- 바이럴 성장 가능성

**전환 전략:**
- 핵심 기능은 무료로 제공하고, 고급 기능은 유료화
- 사용 한도를 설정하고 추가 사용은 유료화
- 광고 제거 옵션 제공

## 수익화 타이밍

수익화를 언제 시작할지는 앱의 성공에 중요한 영향을 미칩니다.

### 초기 단계: 사용자 확보 우선

**제품-시장 적합성 (PMF) 확인:**
수익화를 시도하기 전에 사용자들이 앱의 핵심 가치를 인정하는지 확인하세요. 주요 지표:
- 일간 활성 사용자 (DAU) / 월간 활성 사용자 (MAU) 비율
- 리텐션 율 (D1, D7, D30)
- 사용자 피드백과 평점

**임계 사용자 수 확보:**
충분한 사용자 기반 없이는 광고나 인앱결제 모두 의미 있는 수익을 창출하기 어렵습니다. 최소 수천 명의 활성 사용자를 확보한 후 수익화를 고려하세요.

### 수익화 시작 신호

다음 조건을 충족하면 수익화를 시작할 때입니다:

**높은 참여도:**
- D7 리텐션이 20% 이상
- 사용자가 주 3회 이상 앱을 사용
- 세션 길이가 충분히 길다 (앱 유형에 따라 다름)

**긍정적인 피드백:**
- App Store/Google Play 평점 4.0 이상
- 자발적인 긍정적 리뷰
- 소셜 미디어나 커뮤니티에서의 긍정적 언급

### 점진적 수익화 접근법

**1단계: 비침해적 광고 테스트**
- 보상형 광고부터 시작 (사용자 선택권 제공)
- 사용자 반응 모니터링
- 리텐션에 미치는 영향 측정

**2단계: 인앱결제 도입**
- 광고 제거 옵션 추가
- 프리미엄 기능이나 콘텐츠 제공
- 소비성 아이템 판매 (게임의 경우)

**3단계: 구독 모델 추가**
- 지속적인 가치를 제공할 수 있는 경우
- 정기적인 업데이트나 새 콘텐츠 제공
- 커뮤니티나 지원 서비스 포함

### 수익화 시 피해야 할 실수

**너무 이른 수익화:**
사용자 기반이 확립되기 전에 공격적으로 수익화하면 성장이 저해됩니다.

**과도한 광고:**
사용자 경험을 심각하게 해치는 광고 배치는 이탈을 초래합니다.

**핵심 기능 유료화:**
앱의 기본 가치를 제공하는 핵심 기능은 무료로 유지하고, 추가 가치만 유료화하세요.

### 지표 모니터링

수익화 후 다음 지표를 지속적으로 추적하세요:

**수익 지표:**
- ARPU (Average Revenue Per User)
- ARPPU (Average Revenue Per Paying User)
- 전환율
- LTV (Lifetime Value)

**사용자 지표:**
- 리텐션율 변화
- 이탈률
- 세션 길이 변화
- DAU/MAU 추이

**광고 지표:**
- eCPM
- Fill Rate
- CTR (Click-Through Rate)
- Impressions

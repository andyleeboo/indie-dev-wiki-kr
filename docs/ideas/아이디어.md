# 아이디어 (Ideas)

## 개요

인디 개발에서 아이디어란 단순한 생각을 넘어, '시장 문제와 사용자 필요를 해결할 수 있는 실질적인 해결책'으로 진화하는 과정이다. 좋은 아이디어 자체보다 중요한 것은 빠르고 저렴하게 검증해 시장 적합성을 입증하는 것이다.

## 1. 아이디어 발굴

아이디어는 주변에서 흔히 접하는 불편함, 반복되는 질문, 혹은 인터넷 데이터에서 파생된다. 단순히 '좋아 보인다'는 직관보다, 문제로 느끼는 사람의 수와 해결 의지가 있는지를 살펴야 한다.

### 방법

- 주변 사람/커뮤니티의 문제점 관찰
- Reddit, Indie Hackers 같은 커뮤니티에서 반복되는 질문/요청 참고
- 검색 트렌드 분석
- 경쟁 서비스/시장 갭 분석

## 2. 아이디어 검증의 중요성

아이디어를 제품으로 만들기 전에 검증하는 것은 리소스를 절약하고 실패 리스크를 낮춘다. 경험적으로 9할 이상의 스타트업 실패 사례가 "제품-시장 적합성(PMF)" 실패에서 온다. 따라서 실제 수요를 증명하는 단계가 필수이다.

## 3. 검증 방법

### 1) 고객 인터뷰

잠재 고객의 실제 문제와 니즈를 직접 묻고 확인한다. 솔루션을 제안하기보다 현재의 어려움에 초점을 맞춘다.

### 2) 랜딩페이지 스모크 테스트

제품이나 서비스가 실제로 존재하지 않더라도, 개념 설명 + 가입/관심 등록 버튼을 만들어 클릭률을 본다.

### 3) 사전 판매/입금 테스트

사람들이 실제 돈을 지불할 의향이 있는지를 검증하는 강력한 방법이다.

### 4) 커뮤니티/플랫폼 테스트

Product Hunt, Reddit, Hacker News 같은 플랫폼에 아이디어를 소개하고 반응을 관찰한다.

### 5) 키워드/검색량 검토

검색량이 많은 키워드는 사람들의 수요를 나타내는 신호다.

### 6) 경쟁 분석

기존 경쟁 제품의 사용자 수, 리뷰, 가격 등을 통해 수요와 개선점을 판단한다.

## 4. 검증 지표

- 가입 대기자 수
- 클릭률(CTR) / 전환률
- 사전 구매 의향
- 인터뷰 반복 패턴
- 경쟁 대비 차별화 요소
